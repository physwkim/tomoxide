# tomoxide ‚Äî Roadmap

Milestones from the current scaffold to feature parity. Each milestone is
shippable on its own and keeps `cargo build --workspace` + `cargo nextest run
--workspace` green on a machine with **no** GPU.

## M0 ‚Äî Scaffold ‚úÖ (this commit)

- Workspace, 10 crates, tri-backend trait skeleton (CPU/CUDA/wgpu).
- Data model, geometry, params/enums, errors in `tomoxide-core`.
- Every algorithm/preprocessing entry point exists and dispatches; bodies
  return `Error::NotImplemented` with an upstream `file:line` doc pointer.
- A handful of real CPU ops (`angles`, `minus_log`, `darkflat`, `circ_mask`,
  `shepp2d`) so there is an end-to-end smoke test.
- Design docs: ARCHITECTURE, PORTING, ROADMAP.

**Done = workspace builds and tests pass with the CPU backend selected.**

## M1 ‚Äî CPU analytic core (parity target: tomopy FBP/gridrec) ‚úÖ

FBP and gridrec complete, verified by self-consistent round-trip AND against
checked-in tomopy golden data.

- ‚úÖ `tomoxide-cpu` `Fft::fft_1d` (rustfft, normalized inverse).
- ‚úÖ `FbpFilter`: full padded ramp√ówindow kernel + FFT-domain `apply`.
- ‚úÖ `FilteredBackproject::backproject` (parallel-beam, voxel-driven, rayon).
- ‚úÖ `ForwardProject::project` (parallel-beam, pixel-driven; exact adjoint of
  the back-projector) and `sim::project` wired to it.
- ‚úÖ `Fft::fft_2d` (separable row‚Äìcolumn transforms).
- ‚úÖ `recon::gridrec` ‚Äî Fourier-grid / DFI with a Kaiser‚ÄìBessel gridding kernel
  (gridrec-family, not a PSWF bit-port; backend-agnostic via the `Fft`
  capability).
- ‚úÖ Round-trip gate over a central disk (`crates/tomoxide/tests/`): FBP(ramp)
  Pearson r = 0.96, gridrec r = 0.97.
- ‚úÖ Numeric diff vs tomopy on a fixed shepp2d phantom (tomopy 1.15.3 golden,
  checked in under `crates/tomoxide/tests/fixtures/`, regenerated by
  `tools/gen_tomopy_golden.py`). tomoxide reconstructs tomopy's *own* sinogram:
  - **gridrec** vs tomopy `gridrec`: Pearson r = **0.98** (true cross-impl
    parity).
  - **fbp** vs the phantom: Pearson r = **0.87**. tomopy's own `fbp` is not a
    usable numeric reference here ‚Äî it correlates only r‚âà0.47 with its own
    phantom (every orientation) and r‚âà0.48 with its own gridrec, with a large DC
    offset; tomopy documents `gridrec` as the analytic method. So FBP is gated
    against the phantom ‚Äî recovering it from tomopy's *foreign* sinogram, a
    stronger check than the self-round-trip (it crosses projector conventions).
  - tomopy's reconstruction grid is a fixed vertical (row-axis) flip of
    tomoxide's; the parity test re-indexes by that one convention.

**Done = `fbp`/`gridrec` verified vs tomopy on shepp2d.** gridrec matches tomopy
gridrec to r=0.98; FBP recovers the phantom from tomopy's sinogram to r=0.87.

## M2 ‚Äî CPU iterative family (parity target: tomopy ART/SIRT/MLEM‚Ä¶) üü¢ scalar family done

The full scalar iterative family is implemented and verified (sirt, mlem, osem,
pml/ospml quad & hybrid, grad, tikh, tv, art, bart). Vector tomography
(`vector{,2,3}`, a separate multi-dataset API) is also done ‚Äî see below.


- ‚úÖ `Sirt` ‚Äî R/C-weighted SIRT via the forward/back-projection capabilities;
  round-trip residual decreases monotonically, Pearson r = 0.95.
- ‚úÖ `Mlem` ‚Äî multiplicative EM `x ‚Üê x ‚àò A·µÄ(b‚äòAx) ‚äò A·µÄ(1)`, positivity
  preserved, Pearson r = 0.99.
- ‚úÖ `Osem` ‚Äî MLEM over ordered angle-subsets (`num_block`, `ind_block`); 18
  iters √ó 10 blocks ‚Üí r = 0.99, and `num_block = 1` is bit-identical to MLEM.
- ‚úÖ `PmlQuad`/`OspmlQuad` ‚Äî penalized ML with a quadratic neighbour prior (De
  Pierro update); `reg_par = 0` is bit-identical to MLEM/OSEM, `reg_par > 0`
  smooths (roughness ‚àí42% at r = 0.95). `pml_quad` = `ospml_quad` w/ one block.
- ‚úÖ `PmlHybrid`/`OspmlHybrid` ‚Äî edge-preserving hybrid prior (per-neighbour
  factor `Œ≥ = 1/(1+|Œî|/reg_par[1])`); shares the De Pierro update, so with no
  `reg_par[1]` it is bit-identical to the quadratic prior, and at a small
  threshold it tracks edges far better (r = 0.99 vs quad's 0.90 at equal strength).
- ‚úÖ `Grad` ‚Äî least-squares gradient descent (`grad.c`): gradient
  `g = 2r¬∑R·µÄ(r¬∑R x ‚àí b)`, step `x ‚Üê x ‚àí Œª g`, with a fixed step (`reg_par[0] ‚â• 0`)
  or a Barzilai‚ÄìBorwein self-tuning step (`reg_par[0] < 0`). BB reconstructs the
  full (signed) phantom to r = 0.99. tomopy's `r = 1/‚àö(ncols¬∑nang/2)` step
  normalization is tuned to its Siddon projector, so a unit fixed step diverges
  with this linear-interp adjoint pair (a stably-small fixed step or BB is used);
  same projector-model gap as FBP.
- ‚úÖ `Tikh` ‚Äî Tikhonov-regularized gradient descent (`tikh.c`): `grad` plus a
  ridge gradient `2¬∑reg_par[1]¬∑(x ‚àí reg_data)`. With no `reg_par[1]` (and the
  default zero prior) it is bit-identical to `grad`; a positive weight toward a
  zero prior shrinks the reconstruction energy (449 ‚Üí 102 on the disk) while
  still tracking the phantom. Shares the `gradient_descent` core with `grad`.
- ‚úÖ `Tv` ‚Äî total-variation regularized least squares (`tv.c`), a Chambolle‚ÄìPock
  primal‚Äìdual solver (isotropic TV dual projected to the `reg_par[0]`-ball, L2
  data dual, Œ∏=1 over-relaxation, fixed step `c = 0.35`). Reconstructs the
  piecewise-constant phantom to r = 0.95 (Œª=0.01, 200 iters); a larger Œª smooths
  flat regions (roughness 0.34 ‚Üí 0.22). Stable for this projector across the
  tested Œª/iteration range despite the `grad` normalization caveat.
- ‚úÖ `Art`/`Bart` ‚Äî row-action (Kaczmarz) methods via a new single-ray projector
  capability (`RayProject`, CPU backend): the sparse rows of the same operator
  `R` the other methods use, built once and reused. ART updates after every ray
  (`x ‚Üê x + (b‚àí‚ü®a,x‚ü©)/‚Äña‚Äñ¬≤¬∑a`), r = 0.99 at 20 iters; BART is ordered-subset SART
  (block-simultaneous, `update`/`sum_dist` accumulation applied per block),
  r = 0.98, and more subsets accelerate convergence (residual 689 vs 39792 at 8
  iters, 15 vs 1 block). CUDA/wgpu inherit the `None` accessor until ported.
- ‚úÖ `vector`/`vector2`/`vector3` ‚Äî vector tomography (multi-dataset in,
  vector-field out) in `recon::vector`, a separate API outside the scalar
  `recon()` dispatch. Line-for-line port of tomopy `recon/vector.c` + the
  `utils.c` ray-tracing helpers; bit-exact (Œî=0) vs tomopy 1.15. `vector` takes
  any slice count; `vector2`/`vector3` reconstruct a full `dx¬≥` vector field so
  they require a cube (`dy==dx`) ‚Äî tomopy corrupts memory otherwise, which the
  port rejects explicitly.
- ‚úÖ Block/ordered-subset handling (`num_block`, `ind_block`) ‚Äî `ordered_subsets`
  tiles the angle order into contiguous blocks (tomopy `osem.c`).
- Verification: per-algorithm round-trip + non-negativity; OSEM‚ÜîMLEM and
  penalized‚Üîunpenalized boundaries are bit-identical.

## M3 ‚Äî Preprocessing & center finding (CPU) üü° started

- `prep`: `minus_log`, `normalize*` done; ‚úÖ Paganin `retrieve_phase` (FFT
  low-pass on padded radiographs; matches tomopy to f32 round-off, max rel
  Œî‚âà2.4e-7); ‚úÖ `median_filter3d` + `remove_outlier3d` dezinger (clamp-to-center
  `(2r+1)¬≥` median; bit-exact tomopy parity); ‚úÖ `remove_stripe_sf` (smoothing-
  filter stripe removal; bit-exact tomopy parity); ‚úÖ `remove_ring` (polar-
  transform median/mean ring removal, both `int_mode` = WRAP and REFLECT;
  bit-exact tomopy parity Œî=0); ‚úÖ
  `remove_all_stripe` (Vo algorithms 3+5+6: dead/large-stripe detection +
  bilinear column fill + sorting; matches tomopy to the f32 round-off floor,
  max rel Œî‚âà5.8e-7); ‚úÖ `remove_stripe_ti` (Titarenko/Miqueles: per-slice f64
  finite-difference CG offset solve, `sqrt(d1¬∑d2 + Œ≤¬∑|min|)`; f32-floor parity,
  max rel Œî‚âà5.2e-7; default `nblock=0` only ‚Äî tomopy's block `_ringb` is
  unrunnable on modern numpy). ‚¨ú rest of the stripe family (`fw`).
- `center`: ‚úÖ `find_center_vo` (Nghia Vo, the primary/workhorse) ‚Äî sinogram-
  domain Fourier method (anisotropic-Gaussian denoise ‚Üí double-wedge-masked
  `mean(|fftshift(fft2)|)` metric ‚Üí coarse 0.5-px + fine cubic-B-spline search,
  with the >4e6 column-downsample fast path). Projector-independent, so it
  matches tomopy 1.15.3 **exactly (Œî = 0)** across 4 parity cases ‚Äî true
  cross-impl parity, unlike fbp. ‚úÖ `find_center_pc` (phase-correlation of the
  0¬∞/mirrored-180¬∞ pair ‚Äî a port of skimage `phase_cross_correlation`, phase
  normalization + `1/tol` upsampled-DFT subpixel refinement; projector-
  independent, matches tomopy **exactly (Œî = 0)** across 4 cases incl. two
  subpixel; `rotc_guess` pre-alignment not yet ported). ‚úÖ `find_center`
  (entropy + Nelder-Mead, `rotation.py:82`) ‚Äî reconstructs a slice with gridrec
  at candidate centers and minimises the masked reconstruction's histogram
  entropy via a faithful scalar Nelder-Mead. Projector-coupled (it goes through
  gridrec), so it is held to recovery, not bit parity: it lands on the true axis
  (the projector-independent `find_center_vo`) within ¬±0.5 px and agrees with
  tomopy's own `find_center` within ¬±1 px. Exposed a latent gridrec bug ‚Äî the
  Fourier recentering shift used the raw FFT bin index instead of the signed
  frequency, collapsing reconstructions at sub-pixel centers (invisible at the
  integer default center); fixed (M1 gridrec, bit-identical at integer centers).
  ‚úÖ `write_center`; ‚úÖ `find_center_sift` (OpenCV SIFT, `sift-center` feature).
- `tomoxide-io`: DXchange HDF5 reader/writer + TIFF.

**Done = a full CPU pipeline: HDF in ‚Üí preprocess ‚Üí center ‚Üí FBP ‚Üí TIFF out.**

## M4 ‚Äî CUDA backend (parity target: tomocupy)

- `tomoxide-cuda`: C-ABI shim over tomocupy's `cfunc_*` classes; `build.rs`
  nvcc compile gated on the `cuda` feature; FFI bindings.
- Backends for `fourierrec`, `lprec`, `linerec`, `cfunc_filter`.
- GPU `Elementwise`/stripe/phase to match tomocupy's `proc_functions`.
- Verification: on a CUDA host, numeric diff vs tomocupy for each method.

## M5 ‚Äî Streaming pipeline (parity target: tomocupy `rec_steps`)

- `pipeline::ReconSteps`: sinogram/projection chunking, double buffering,
  3-stage overlap (read ‚Üí H2D ‚Üí compute ‚Üí D2H ‚Üí write), read/write thread
  pools, `try`/`try_lamino` center & laminography sweeps.
- Out-of-core reconstruction of a dataset larger than device memory.

## M6 ‚Äî Portable GPU (wgpu / Metal)

- `tomoxide-wgpu`: WGSL ports of the FBP filter, backprojection, elementwise.
- Runs the GPU path on Apple Silicon (Metal) and Vulkan/DX12.
- Verification: wgpu vs CPU numeric diff on the same phantom.

## M7 ‚Äî Laminography, beam hardening, AI center, polish

- USFFT laminography (`lamfourierrec`, `cfunc_usfft1d/2d`, `cfunc_fft2d`).
- Beam-hardening correction; SIFT/AI center finding.
- `f16` paths on GPU backends; zarr output; benchmarks (`hyperfine`/criterion).

### Cross-cutting verification harness

A `tests/` parity harness (added in M1) fixes a small phantom, runs the
reference (tomopy/tomocupy via a checked-in `.npy`/`.h5` golden), and asserts
tomoxide is within tolerance ‚Äî backend-parametrized so CPU/CUDA/wgpu all run
the same assertions.
