# tomoxide тАФ Roadmap

Milestones from the current scaffold to feature parity. Each milestone is
shippable on its own and keeps `cargo build --workspace` + `cargo nextest run
--workspace` green on a machine with **no** GPU.

## M0 тАФ Scaffold тЬЕ (this commit)

- Workspace, 10 crates, tri-backend trait skeleton (CPU/CUDA/wgpu).
- Data model, geometry, params/enums, errors in `tomoxide-core`.
- Every algorithm/preprocessing entry point exists and dispatches; bodies
  return `Error::NotImplemented` with an upstream `file:line` doc pointer.
- A handful of real CPU ops (`angles`, `minus_log`, `darkflat`, `circ_mask`,
  `shepp2d`) so there is an end-to-end smoke test.
- Design docs: ARCHITECTURE, PORTING, ROADMAP.

**Done = workspace builds and tests pass with the CPU backend selected.**

## M1 тАФ CPU analytic core (parity target: tomopy FBP/gridrec) тЬЕ

FBP and gridrec complete, verified by self-consistent round-trip AND against
checked-in tomopy golden data.

- тЬЕ `tomoxide-cpu` `Fft::fft_1d` (rustfft, normalized inverse).
- тЬЕ `FbpFilter`: full padded ramp├Чwindow kernel + FFT-domain `apply`.
- тЬЕ `FilteredBackproject::backproject` (parallel-beam, voxel-driven, rayon).
- тЬЕ `ForwardProject::project` (parallel-beam, pixel-driven; exact adjoint of
  the back-projector) and `sim::project` wired to it.
- тЬЕ `Fft::fft_2d` (separable rowтАУcolumn transforms).
- тЬЕ `recon::gridrec` тАФ Fourier-grid / DFI with a KaiserтАУBessel gridding kernel
  (gridrec-family, not a PSWF bit-port; backend-agnostic via the `Fft`
  capability).
- тЬЕ Round-trip gate over a central disk (`crates/tomoxide/tests/`): FBP(ramp)
  Pearson r = 0.96, gridrec r = 0.97.
- тЬЕ Numeric diff vs tomopy on a fixed shepp2d phantom (tomopy 1.15.3 golden,
  checked in under `crates/tomoxide/tests/fixtures/`, regenerated by
  `tools/gen_tomopy_golden.py`). tomoxide reconstructs tomopy's *own* sinogram:
  - **gridrec** vs tomopy `gridrec`: Pearson r = **0.98** (true cross-impl
    parity).
  - **fbp** vs the phantom: Pearson r = **0.87**. tomopy's own `fbp` is not a
    usable numeric reference here тАФ it correlates only rтЙИ0.47 with its own
    phantom (every orientation) and rтЙИ0.48 with its own gridrec, with a large DC
    offset; tomopy documents `gridrec` as the analytic method. So FBP is gated
    against the phantom тАФ recovering it from tomopy's *foreign* sinogram, a
    stronger check than the self-round-trip (it crosses projector conventions).
  - tomopy's reconstruction grid is a fixed vertical (row-axis) flip of
    tomoxide's; the parity test re-indexes by that one convention.

**Done = `fbp`/`gridrec` verified vs tomopy on shepp2d.** gridrec matches tomopy
gridrec to r=0.98; FBP recovers the phantom from tomopy's sinogram to r=0.87.

## M2 тАФ CPU iterative family (parity target: tomopy ART/SIRT/MLEMтАж) ЁЯЯв scalar family done

The full scalar iterative family is implemented and verified (sirt, mlem, osem,
pml/ospml quad & hybrid, grad, tikh, tv, art, bart). Vector tomography
(`vector{,2,3}`, a separate multi-dataset API) is also done тАФ see below.


- тЬЕ `Sirt` тАФ R/C-weighted SIRT via the forward/back-projection capabilities;
  round-trip residual decreases monotonically, Pearson r = 0.95.
- тЬЕ `Mlem` тАФ multiplicative EM `x тЖР x тИШ Aс╡А(bтКШAx) тКШ Aс╡А(1)`, positivity
  preserved, Pearson r = 0.99.
- тЬЕ `Osem` тАФ MLEM over ordered angle-subsets (`num_block`, `ind_block`); 18
  iters ├Ч 10 blocks тЖТ r = 0.99, and `num_block = 1` is bit-identical to MLEM.
- тЬЕ `PmlQuad`/`OspmlQuad` тАФ penalized ML with a quadratic neighbour prior (De
  Pierro update); `reg_par = 0` is bit-identical to MLEM/OSEM, `reg_par > 0`
  smooths (roughness тИТ42% at r = 0.95). `pml_quad` = `ospml_quad` w/ one block.
- тЬЕ `PmlHybrid`/`OspmlHybrid` тАФ edge-preserving hybrid prior (per-neighbour
  factor `╬│ = 1/(1+|╬Ф|/reg_par[1])`); shares the De Pierro update, so with no
  `reg_par[1]` it is bit-identical to the quadratic prior, and at a small
  threshold it tracks edges far better (r = 0.99 vs quad's 0.90 at equal strength).
- тЬЕ `Grad` тАФ least-squares gradient descent (`grad.c`): gradient
  `g = 2r┬╖Rс╡А(r┬╖R x тИТ b)`, step `x тЖР x тИТ ╬╗ g`, with a fixed step (`reg_par[0] тЙе 0`)
  or a BarzilaiтАУBorwein self-tuning step (`reg_par[0] < 0`). BB reconstructs the
  full (signed) phantom to r = 0.99. tomopy's `r = 1/тИЪ(ncols┬╖nang/2)` step
  normalization is tuned to its Siddon projector, so a unit fixed step diverges
  with this linear-interp adjoint pair (a stably-small fixed step or BB is used);
  same projector-model gap as FBP.
- тЬЕ `Tikh` тАФ Tikhonov-regularized gradient descent (`tikh.c`): `grad` plus a
  ridge gradient `2┬╖reg_par[1]┬╖(x тИТ reg_data)`. With no `reg_par[1]` (and the
  default zero prior) it is bit-identical to `grad`; a positive weight toward a
  zero prior shrinks the reconstruction energy (449 тЖТ 102 on the disk) while
  still tracking the phantom. Shares the `gradient_descent` core with `grad`.
- тЬЕ `Tv` тАФ total-variation regularized least squares (`tv.c`), a ChambolleтАУPock
  primalтАУdual solver (isotropic TV dual projected to the `reg_par[0]`-ball, L2
  data dual, ╬╕=1 over-relaxation, fixed step `c = 0.35`). Reconstructs the
  piecewise-constant phantom to r = 0.95 (╬╗=0.01, 200 iters); a larger ╬╗ smooths
  flat regions (roughness 0.34 тЖТ 0.22). Stable for this projector across the
  tested ╬╗/iteration range despite the `grad` normalization caveat.
- тЬЕ `Art`/`Bart` тАФ row-action (Kaczmarz) methods via a new single-ray projector
  capability (`RayProject`, CPU backend): the sparse rows of the same operator
  `R` the other methods use, built once and reused. ART updates after every ray
  (`x тЖР x + (bтИТтЯиa,xтЯй)/тАЦaтАЦ┬▓┬╖a`), r = 0.99 at 20 iters; BART is ordered-subset SART
  (block-simultaneous, `update`/`sum_dist` accumulation applied per block),
  r = 0.98, and more subsets accelerate convergence (residual 689 vs 39792 at 8
  iters, 15 vs 1 block). CUDA/wgpu inherit the `None` accessor until ported.
- тЬЕ `vector`/`vector2`/`vector3` тАФ vector tomography (multi-dataset in,
  vector-field out) in `recon::vector`, a separate API outside the scalar
  `recon()` dispatch. Line-for-line port of tomopy `recon/vector.c` + the
  `utils.c` ray-tracing helpers; bit-exact (╬Ф=0) vs tomopy 1.15. `vector` takes
  any slice count; `vector2`/`vector3` reconstruct a full `dx┬│` vector field so
  they require a cube (`dy==dx`) тАФ tomopy corrupts memory otherwise, which the
  port rejects explicitly.
- тЬЕ Block/ordered-subset handling (`num_block`, `ind_block`) тАФ `ordered_subsets`
  tiles the angle order into contiguous blocks (tomopy `osem.c`).
- Verification: per-algorithm round-trip + non-negativity; OSEMтЖФMLEM and
  penalizedтЖФunpenalized boundaries are bit-identical.

## M3 тАФ Preprocessing & center finding (CPU) ЁЯЯб started

- `prep`: `minus_log`, `normalize*` done; тЬЕ Paganin `retrieve_phase` (FFT
  low-pass on padded radiographs; matches tomopy to f32 round-off, max rel
  ╬ФтЙИ2.4e-7); тЬЕ `median_filter3d` + `remove_outlier3d` dezinger (clamp-to-center
  `(2r+1)┬│` median; bit-exact tomopy parity); тЬЕ `remove_stripe_sf` (smoothing-
  filter stripe removal; bit-exact tomopy parity); тЬЕ `remove_ring` (polar-
  transform median/mean ring removal, both `int_mode` = WRAP and REFLECT;
  bit-exact tomopy parity ╬Ф=0); тЬЕ
  `remove_all_stripe` (Vo algorithms 3+5+6: dead/large-stripe detection +
  bilinear column fill + sorting; matches tomopy to the f32 round-off floor,
  max rel ╬ФтЙИ5.8e-7); тЬЕ `remove_stripe_ti` (Titarenko/Miqueles: per-slice f64
  finite-difference CG offset solve, `sqrt(d1┬╖d2 + ╬▓┬╖|min|)`; f32-floor parity,
  max rel ╬ФтЙИ5.2e-7; default `nblock=0` only тАФ tomopy's block `_ringb` is
  unrunnable on modern numpy). тмЬ rest of the stripe family (`fw`).
- `center`: тЬЕ `find_center_vo` (Nghia Vo, the primary/workhorse) тАФ sinogram-
  domain Fourier method (anisotropic-Gaussian denoise тЖТ double-wedge-masked
  `mean(|fftshift(fft2)|)` metric тЖТ coarse 0.5-px + fine cubic-B-spline search,
  with the >4e6 column-downsample fast path). Projector-independent, so it
  matches tomopy 1.15.3 **exactly (╬Ф = 0)** across 4 parity cases тАФ true
  cross-impl parity, unlike fbp. тЬЕ `find_center_pc` (phase-correlation of the
  0┬░/mirrored-180┬░ pair тАФ a port of skimage `phase_cross_correlation`, phase
  normalization + `1/tol` upsampled-DFT subpixel refinement; projector-
  independent, matches tomopy **exactly (╬Ф = 0)** across 4 cases incl. two
  subpixel; `rotc_guess` pre-alignment not yet ported). тЬЕ `find_center`
  (entropy + Nelder-Mead, `rotation.py:82`) тАФ reconstructs a slice with gridrec
  at candidate centers and minimises the masked reconstruction's histogram
  entropy via a faithful scalar Nelder-Mead. Projector-coupled (it goes through
  gridrec), so it is held to recovery, not bit parity: it lands on the true axis
  (the projector-independent `find_center_vo`) within ┬▒0.5 px and agrees with
  tomopy's own `find_center` within ┬▒1 px. Exposed a latent gridrec bug тАФ the
  Fourier recentering shift used the raw FFT bin index instead of the signed
  frequency, collapsing reconstructions at sub-pixel centers (invisible at the
  integer default center); fixed (M1 gridrec, bit-identical at integer centers).
  тЬЕ `write_center`; тЬЕ `find_center_sift` (OpenCV SIFT, `sift-center` feature).
- `tomoxide-io`: DXchange HDF5 reader/writer + TIFF.

**Done = a full CPU pipeline: HDF in тЖТ preprocess тЖТ center тЖТ FBP тЖТ TIFF out.**

## M4 тАФ CUDA backend (parity target: tomocupy) ЁЯЯв FBP back-projection live

- тЬЕ `tomoxide-cuda`: C-ABI shim over tomocupy's `cfunc_*` classes + minimal CUDA
  runtime helpers (device probe, `cudaMalloc`/memcpy/free); `build.rs` nvcc
  compile of the vendored kernels gated on the `cuda` feature; FFI bindings.
  `CudaBackend::new` probes `cudaGetDeviceCount`.
- тЬЕ `linerec` parallel-beam back-projection on the GPU (`cfunc_linerec`),
  exposed as the `FilteredBackproject` capability. The FBP **filter** reuses the
  shared CPU definition (`CudaBackend::fbp_filter` тЖТ `CpuBackend`), so
  `recon(Fbp, &CudaBackend)` filters on the host and back-projects on the device.
  Verified on an RTX 5000 Ada: CUDAтЖФCPU back-projection Pearson = 1.00000 (up to
  the kernel's y-flip + `4/nproj` vs `╧А/nproj` scale), CUDAтЖФphantom 0.96
  (`crates/tomoxide/tests/cuda_fbp_parity.rs`). Needs тЙе2 slices (the kernel
  interpolates vertically).
- тмЬ `fourierrec`/`lprec` GPU back-projection (cufft, the complex slice-packing),
  GPU `cfunc_filter`, and GPU `Elementwise`/stripe/phase to match tomocupy's
  `proc_functions`.
- Verification: on a CUDA host, numeric diff vs the CPU backend per method.
